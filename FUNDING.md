# Funding Engram

Engram is a live testnet Bittensor subnet for permanent, content-addressed AI
memory. It stores semantic vectors across miners, verifies storage with proofs,
and archives raw media through Arweave.

## What Funding Unlocks

Near-term funding is focused on making the subnet reliable enough for mainnet
and serious external users:

- Security hardening for namespace auth, private memory, request signing, and
  storage proofs.
- Miner and validator operations on Bittensor testnet and mainnet.
- Rust core work for faster, safer proof and CID primitives.
- Arweave-backed media permanence for images, PDFs, URLs, and public datasets.
- SDK integrations for AI agents, LangChain, LlamaIndex, and retrieval systems.
- Documentation, tutorials, and onboarding for miners, validators, and users.

## Good Fits

Engram is a good fit for support from:

- Bittensor validators, subnet operators, miners, stakers, and ecosystem funds.
- Arweave and permanent-storage ecosystem builders.
- Open-source and public-goods grant programs.
- AI agent teams that need portable, persistent, verifiable memory.
- Security, cryptography, Rust, and distributed-systems contributors.

## Ways To Support

- Sponsor a milestone such as mainnet readiness, proof hardening, encrypted
  namespaces, replication repair, or SDK integrations.
- Run a miner or validator and help test subnet economics.
- Integrate Engram into an agent, RAG product, archive, or research workflow.
- Provide infrastructure credits for validators, miners, storage, monitoring,
  CI, and security review.
- Fund contributor bounties for well-scoped GitHub issues.

## Priority Milestones

| Milestone | Why it matters |
| --- | --- |
| Mainnet readiness | Makes Engram credible for miners, validators, stakers, and users. |
| Storage proof hardening | Strengthens the core trust claim of verifiable AI memory. |
| Threshold private namespaces | Reduces trust in any single miner for private memory. |
| Runtime replication repair | Keeps CIDs available when miners churn or fail. |
| Production web signing | Prevents unauthenticated writes through public API routes. |
| Package and license cleanup | Removes blockers for grants, contributors, and downstream use. |
